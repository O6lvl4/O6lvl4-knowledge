# Audit Relations

vault 内のノート間リレーションを点検・整理する。

## スコープ

`$ARGUMENTS`

- ファイルパス指定 (例: `vault/wasm-core.md`) → そのノートの関連を点検
- タグ指定 (例: `webassembly`) → そのタグを持つノート群を横断点検
- `all` または空 → vault 全体を点検

## 手順

### 1. リンクの収集

対象ノートから以下を収集する:

- `[[wikilink]]` の一覧 (関連セクション + 本文中)
- 他ノートから対象ノートへの逆リンク (Grep で `\[\[ノート名\]\]` を検索)
- index.md でのエントリ

### 2. 問題の検出

以下の観点で各リンクを評価する:

**壊れたリンク (Broken)**
- `[[target]]` の target に対応する `vault/target.md` が存在しない
- → 削除するか、正しいノート名に修正

**無理やりなリンク (Forced)**
- リンク先の内容と実質的な関連がない (名前が似ているだけ、間接的すぎる)
- 関連セクションの説明文が曖昧・こじつけ (「〜に関連」としか書けない程度の関係)
- 判定基準: そのリンクを辿った読者が「なるほど、だからリンクされてるのか」と納得できるか
- → 削除、または説明文を書き直して関連性を明確化

**欠けているリンク (Missing)**
- 内容的に明らかに関連するノートが vault にあるのにリンクされていない
- 双方向リンクの片方だけ存在する (A→B はあるが B→A がない)
- → 追加 (ただし本当に有用な接続だけ。数を増やすことが目的ではない)

**冗長なリンク (Redundant)**
- 同一ノートへの `[[wikilink]]` が本文中に何度も出現 (初出以外は不要)
- 関連セクションと本文中で同じリンクが重複し、関連セクション側の説明が本文の繰り返し
- → 本文中は初出のみ残す。関連セクションは独自の文脈説明があるなら残す

**孤立ノート (Orphan)**
- どのノートからもリンクされていない
- index.md にも載っていない
- → リンクを追加するか、ユーザーに報告

### 3. 修正の実行

問題を一覧にして報告した後、修正を実行する:

- 壊れたリンク → 即座に修正/削除
- 無理やりなリンク → 削除、理由を報告
- 欠けているリンク → 追加、なぜ必要かを説明
- 冗長なリンク → 整理
- 孤立ノート → ユーザーに判断を仰ぐ

### 4. 報告

点検結果を以下の形式で簡潔に報告する:

```
## 点検結果: [スコープ]

対象ノート数: N
検出した問題: N件

### 修正済み
- [ノート名]: 壊れたリンク [[xxx]] を削除
- [ノート名]: [[yyy]] → [[zzz]] の逆リンクを追加

### 削除済み (無理やりなリンク)
- [ノート名] → [[xxx]]: 理由

### 要確認
- [ノート名]: 孤立ノート。リンク先候補: ...
```
