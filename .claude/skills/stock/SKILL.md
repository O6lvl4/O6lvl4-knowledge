# Stock Knowledge

ユーザーが投げた自然言語を vault のナレッジとして蓄積する。

## 入力

`$ARGUMENTS`

入力がない場合は、直前のユーザーメッセージの内容を対象とする。

## 手順

### 1. 入力の分析

投げられたテキストから以下を抽出する:

- **トピック**: 何についての知識か (1つとは限らない。複数トピックにまたがる場合がある)
- **知識の種類**: 概念の説明 / 比較・トレードオフ / 実装パターン / 設計判断の理由 / 事実・数値 / 意見・洞察
- **知識密度**: 1ノートに十分な量か、既存ノートへの追記で済むか

### 2. 既存ノートの検索

vault 内を Grep/Glob で検索し、関連する既存ノートを探す:

- ファイル名での一致 (`vault/*topic*.md`)
- 内容での一致 (キーワードで Grep)
- tags での一致
- `[[wikilink]]` での参照関係

### 3. 判断: 追記 vs 新規作成

**既存ノートに追記する場合:**
- トピックが既存ノートのスコープ内に収まる
- 既存ノートに新しい情報を追加する形が自然
- → 既存ノートの適切なセクションに追記、または新セクションを追加

**新規ノートを作成する場合:**
- 既存ノートにトピックが見つからない
- 十分な知識量があり、独立したノートとして成立する
- → `/enrich` スキルの品質基準に従って新規ノート作成

**複数ノートにまたがる場合:**
- 各ノートに該当部分を振り分ける
- ノート間の `[[wikilink]]` を追加

### 4. 書き込みルール

- **ユーザーの言葉を咀嚼して再構成する** — コピペではなく、vault の文体に合わせて構造化する
- **知識密度を維持する** — 冗長な説明、前置き、感想は削ぎ落とす。本質だけ残す
- **vault の文体に合わせる**:
  - 冒頭パラグラフは「何であるか」を1-2文で
  - コード例があるなら残す
  - 比較はテーブルで
  - 「なぜ」を大事にする
- **出典の扱い**: ユーザーが URL や書籍名を含めていたら Links セクションに入れる
- **updated_at を現在時刻に更新する** — 日付だけでなく時刻まで含めた ISO 8601 (例: `2026-05-29T15:03:37+09:00`)。recent ビューは同じ日の中ではこの時刻で並べ替えるため、時刻が無いと日内の順序が壊れる。時刻は `date +%H:%M:%S` 等で取得する

### 5. インデックスと相互リンク

- 新規ノート作成時: `vault/index.md` の適切なタグセクションにエントリを追加
- 関連する既存ノートの「関連」セクションに `[[wikilink]]` を追加 (双方向)

### 6. 報告

完了後、以下を簡潔に報告する:

- 何をしたか (追記 / 新規作成)
- どのノートに書いたか
- 追加した知識の要約 (1-2行)
