映像・音声ソースを合成し、配信プラットフォームやファイルに出力するソフトウェアのジャンル。

## このジャンルを定義するプロトコル要素

「配信ソフト」と呼ばれるために実装すべきインターフェースは概ね以下の3層に分かれる。

### 1. 入力（Source）

何を取り込めるか。

| プロトコル / 形式 | 内容 |
|---|---|
| カメラデバイス | USB / 内蔵カメラ（V4L2, AVFoundation, DirectShow） |
| 画面キャプチャ | デスクトップ全体 or 個別ウィンドウ |
| 仮想カメラ入力 | 他アプリが仮想カメラとして映像を渡す（VTuber アプリはここに接続） |
| NDI | ネットワーク経由の映像入力。LAN 内で低遅延 |
| メディアファイル | 画像、動画、音声ファイル |
| ブラウザソース | URL を指定して Web ページを映像として取り込む（チャットオーバーレイ等） |
| キャプチャカード | HDMI 入力をキャプチャ（ゲーム機、外部カメラ） |

### 2. 合成（Scene / Composition）

取り込んだソースをどう組み合わせるか。

- **レイヤリング** — ソースを重ねる。Z 順序で前後関係を制御
- **シーン切り替え** — 複数の構成をプリセットして瞬時に切り替え（トランジション付き）
- **オーディオミキシング** — 複数音声ソースの音量、フィルタ（ノイズ抑制、コンプレッサー）
- **フィルタ** — クロマキー（グリーンバック合成）、色補正、LUT

### 3. 出力（Output）

どこに送り出すか。

| プロトコル | 用途 |
|---|---|
| **RTMP / RTMPS** | 配信の事実上の標準。YouTube, Twitch, ニコ生 がすべて対応 |
| **SRT** | 低遅延・高信頼性。不安定な回線向き |
| **WHIP / WHEP** | WebRTC ベース。超低遅延（< 1秒）。Cloudflare Stream 等 |
| **HLS / DASH** | セグメントベース。CDN 配信向き。遅延は大きい（5-30秒） |
| **録画** | MP4 / MKV / FLV へのローカル保存 |
| **仮想カメラ出力** | 合成結果を仮想カメラデバイスとして他アプリ（Zoom, Discord）に渡す |

### エンコーダ

映像を圧縮する部分。これがリアルタイム配信のボトルネック。

| エンコーダ | 種別 | プラットフォーム |
|---|---|---|
| x264 | CPU (ソフトウェア) | すべて |
| NVENC | GPU (NVIDIA) | Windows, Linux |
| VideoToolbox | GPU (Apple) | macOS |
| QuickSync | GPU (Intel) | Windows, Linux |
| AMF | GPU (AMD) | Windows |

コーデックは H.264 が最も互換性が高い。H.265 (HEVC) と AV1 は高圧縮率だがプラットフォーム対応がまだ限定的。

## VTuber ワークフローにおける位置

```
[[face-tracking|フェイストラッキング]]
    ↓
VTuber アプリ（[[3tene]] 等）
    ↓ 仮想カメラ出力
配信ソフト（[[obs|OBS Studio]] 等）  ← ゲーム画面、チャット、アラートも合成
    ↓ RTMP
YouTube / Twitch
```

配信ソフトは VTuber パイプラインの「出口」。アバター映像を他のソースと合成して配信プラットフォームに送る。

## 代表的なソフトウェア

- [[obs|OBS Studio]] — オープンソース。事実上の標準
- Streamlabs — OBS ベースにオーバーレイ・アラート統合
- XSplit — 商用。Windows 向け
- vMix — プロ向け。マルチカメラ、リプレイ
