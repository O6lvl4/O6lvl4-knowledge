ビジネスの採算を「1人の顧客」単位で見る2つの指標。

- **CAC（Customer Acquisition Cost / 顧客獲得単価）**: 顧客1人を獲得するのにかかった費用。`CAC = 獲得に使った総コスト ÷ 獲得した顧客数`（広告費・営業人件費・販促費を含む）。
- **LTV（Lifetime Value / 顧客生涯価値）**: 1人の顧客が取引期間全体でもたらす**粗利**の総額（売上ではなく利益で見るのが正しい）。

## 成立条件 — LTV > CAC

獲得に払った額を、その顧客から得る利益が上回って初めて事業が成立する。

```
成立:  LTV > CAC
健全:  LTV / CAC ≳ 3       （SaaS の経験則。獲得・運用・利益のバランスの目安）
回収:  CAC 回収期間 < 12〜18ヶ月  （払った CAC を粗利で取り戻すまで）
```

- **LTV/CAC ≈ 1**: 顧客を獲るたびトントンで、利益も成長原資も残らない。
- **LTV/CAC が高すぎ（例 >5）**: 黒字だが**獲得に投資不足**＝もっと攻めれば伸ばせるシグナルでもある。

### LTV の素直な式（サブスク）

```
LTV = ARPU × 粗利率 × (1 / チャーン率)
```

`1 / 月次チャーン率` が平均継続月数。月チャーン 5% なら平均20ヶ月。**チャーンを下げる＝継続を伸ばす**ことが LTV を最も強く動かす（→[[lock-in]]）。

## 無料配布／フリーミアムとの接続

- **無料配布は現物で払う CAC**: [[free-distribution-strategy]]の「N本無料」は、広告費の代わりに**現物（商品原価）で支払う顧客獲得コスト**。広告より確実に「最初の体験」まで到達させられる。
- **高頻度・低単価・反復購買では LTV が習慣でほぼ決まる**: 自販機のような商材は、1人の LTV が「習慣化したか否か」でほぼ決定する。だから習慣形成に直結する現物 CAC は、認知止まりの広告 CAC より効率がよいことがある。
- **[[freemium]]では無料運用コスト ≒ CAC**: 無料ユーザーを支えるサーバ・サポート費は、転換した有料顧客の LTV で回収する構造。

## 落とし穴

- **売上を LTV と誤る**: 粗利で計算しないと過大評価。原価・サポート費を引く。
- **CAC に間接費を入れ忘れる**: 営業人件費・ツール費・無料層の運用費まで含めないと過小評価。
- **平均で語る**: チャネル別・コホート別で LTV/CAC は大きく違う。良いチャネルに寄せる判断は分解しないと出ない。
