---
title: Type 0 文法（句構造文法）
tags: [formal-language, automata-theory, computer-science]
---

[[chomsky-hierarchy|チョムスキー階層]]の最上位。生成規則に制約がない文法。

## 定義

生成規則: α → β

- α は非終端記号を1つ以上含む任意の記号列
- β は任意の記号列（空文字列 ε も可）

左辺の長さが右辺より大きくてもよい（文字列が縮む規則を許容する）。

## 認識する機械: チューリングマシン

無限のテープと読み書きヘッドを持つ計算モデル。Type 0 文法が生成する言語 = チューリングマシンが認識できる言語 = **再帰的可算言語**。

ただし「認識できる」は「受理する入力には必ず停止する」ではなく、「受理する入力には accept を返す」だけ。拒否する入力では永遠にループする可能性がある。

## 決定不能性

Type 0 言語の所属判定（「この文字列はこの言語に属するか？」）は一般に決定不能。停止問題と等価。

## 例

```
G = (N, Σ, P, S)
N = { S, A, B, C }
Σ = { a, b, c }
P = {
  S  → aAbc,
  A  → aAbC,
  A  → ε,
  Cb → bC,
  Cc → cc
}
```

この文法は a^n b^n c^n を生成する…が、Type 1 でも記述可能な言語なので、Type 0 固有の例としては適切ではない。Type 0 でしか記述できない言語は、チューリングマシンの停止問題に関連するような、より複雑なもの。

## 実用での位置づけ

Type 0 が直接使われることはほぼない。理論的に「計算可能性の限界」を示す枠組みとして重要。
