[[chomsky-hierarchy|チョムスキー階層]]の Type 1。生成規則が「文字列を縮めない」という制約を持つ。

## 定義

生成規則: αAβ → αγβ

- A は非終端記号
- α, β は任意の記号列（文脈）
- γ は空でない記号列

核心: A を γ に置換するが、**周囲の文脈 α と β が一致するときだけ**適用できる。これが「文脈依存」の由来。

等価な制約として |左辺| ≤ |右辺|（非短縮的）とも表現できる。

## 認識する機械: 線形有界オートマトン (LBA)

チューリングマシンの制限版。テープの使用領域が入力長に比例する定数倍以内に制限される。

## 例: a^n b^n c^n

```
G = (N, Σ, P, S)
N = { S, B, C }
Σ = { a, b, c }
P = {
  S  → aSBC,
  S  → aBC,
  CB → BC,
  aB → ab,
  bB → bb,
  bC → bc,
  cC → cc
}
```

a^n b^n c^n は Type 2（文脈自由文法）では記述不能だが、Type 1 では記述できる。3つの記号の個数を同時に揃える必要があり、スタック1本では追跡しきれないため。

## Type 2 との境界

- a^n b^n → Type 2 で記述可能（スタック1本で n をカウントできる）
- a^n b^n c^n → Type 1 が必要（2つの量を同時に追跡する必要がある）

## 実用での位置づけ

自然言語の一部の構造（スイス・ドイツ語の交差依存構造など）は文脈依存的とされる。プログラミング言語では、型チェックや名前解決など構文解析「後」の処理が文脈依存にあたる。
