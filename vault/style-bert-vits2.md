自分の音声データからカスタム音声モデルを学習できるローカル TTS ソフトウェア。[[voicevox|VOICEVOX]] がプリセットキャラのみなのに対し、Style-Bert-VITS2 は**オリジナルの声**を作れるのが最大の差別化点。

## 何をするものか

数分〜数十分の音声データを入力として、その声で任意のテキストを読み上げるモデルを学習する。学習後は API サーバーとして動作し、テキストを送ると音声を返す。

## 学習に必要なもの

| 項目 | 要件 |
|---|---|
| 音声データ | 5〜30 分程度（多いほど品質向上） |
| 前処理 | 句ごとに分割 + テキスト書き起こし（自動化ツールあり） |
| GPU | VRAM 8GB 以上推奨 |
| 学習時間 | 数時間（GPU 性能依存。1 回やれば完了） |
| 環境 | Python（セットアップは VOICEVOX より手間がかかる） |

## 音声品質

- **感情表現・イントネーション**が自然。VOICEVOX より表現力が高いと評される
- **日本語に特化**しており精度が高い
- スタイル（感情）の制御が可能

## 推論速度

| 環境 | レイテンシ |
|---|---|
| GPU あり | 100〜300ms（VOICEVOX と同等） |
| CPU のみ | 遅い（実用的ではない） |

API サーバーとして立てられるので、AITuber アプリから HTTP 経由で呼べる。VOICEVOX と同じ使い方が可能。

## VOICEVOX との比較

| | VOICEVOX | Style-Bert-VITS2 |
|---|---|---|
| 音声 | プリセットキャラのみ（20 種以上） | カスタム音声を学習可能 |
| セットアップ | インストールだけ | Python 環境 + 学習パイプライン |
| 学習コスト | なし | 音声データ準備 + 数時間の学習 |
| 推論速度 (GPU) | 100〜300ms | 100〜300ms（同等） |
| ランニングコスト | ¥0 | ¥0 |
| API | あり（localhost:50021） | あり（HTTP） |
| 音質 | 高い | 高い（表現力はやや上） |

## 他のカスタム音声の選択肢

| 方法 | 仕組み | 特徴 |
|---|---|---|
| **Style-Bert-VITS2** | 音声データから直接 TTS モデルを学習 | 最も直接的。品質が高い |
| **RVC + VOICEVOX** | VOICEVOX で合成 → RVC でリアルタイム声質変換 | 既存キャラの発話を別の声に変える。レイテンシ増加あり |
| **COEIROINK** | VOICEVOX フォーク。自作音声モデルの追加に対応 | API 互換あり。選択肢としてはマイナー |

## AITuber 用途での現実的な導入パス

1. まず **VOICEVOX のプリセットキャラ**で AITuber を動かす（即座に開始可能）
2. オリジナル声が必要になったら **Style-Bert-VITS2 で学習**
3. API エンドポイントを差し替えるだけで移行

いきなり Style-Bert-VITS2 から入ると学習データ準備のハードルがある。VOICEVOX で配信パイプラインを確立してから声を差し替えるほうが手堅い。

## 押さえどころ（カード化候補）

- Style-Bert-VITS2 の核心 → **自分の音声データ（5〜30 分）からカスタム TTS モデルを学習できるローカル TTS。VOICEVOX がプリセットのみなのに対し、オリジナル声を作れる**
- 学習要件 → **音声データ 5〜30 分 + GPU VRAM 8GB 以上 + 数時間の学習。1 回やれば完了**
- 推論速度 → **GPU ありで 100〜300ms。VOICEVOX と同等のレイテンシ。API サーバーとして動作**
- VOICEVOX との使い分け → **プリセットキャラで十分なら VOICEVOX。オリジナル声が必要なら Style-Bert-VITS2。両方ローカル実行で TTS コスト ¥0**
