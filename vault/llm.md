膨大な文章を学習して、人間のように言葉を理解・生成できる巨大なモデル。質問に答えたり、文章を書いたり、コードを生成したりできる。

## 何ができる？／なぜ重要？

世界中の本や記事をひたすら読み続けた「物知りな図書館員」を想像してください。どんな話題を振っても、それらしい答えを返してくれる相手です。LLM はこの図書館員をプログラムにしたもので、質問に答えたり、要約を作ったり、メールの下書きを書いたり、プログラムを書いたりできます。

これが嬉しいのは、これまで「人間にしかできない」と思われていた知的作業の多くを、機械にお願いできるようになったからです。なければ、私たちは情報を一つずつ手で探し、文章を一行ずつ自分で書く必要があります。LLM があると、全体の方針を伝えるだけで「下書き」をすぐに用意してくれるので、判断や仕上げに集中できます。

## 仕組み

```mermaid
flowchart LR
    入力[文章を入力] --> トークン化[トークンに分解]
    トークン化 --> モデル[巨大なモデル<br/>数千億の数値]
    モデル --> 推論[次に来る言葉を予測]
    推論 --> 出力[文章を組み立てる]
    出力 --> 結果[回答を返す]
```

入力された文章はまず細かい単位（トークン）に切り分けられ、モデルが「この続きに最もありそうな言葉」を一つずつ予測していきます。これを繰り返して、自然な文章ができあがります。

## 用語

- **モデル**: 学習結果を保存した「脳みそ」のような数値の集まり。
- **学習 (Training)**: 大量の文章を読み込んでパターンを覚えさせる工程。
- **推論 (Inference)**: 学習済みモデルが実際に答えを生成する工程。
- **トークン**: 文章を機械が扱う最小単位。日本語だと1〜2文字程度。
- **コンテキストウィンドウ**: 一度に覚えていられる文章の量の上限。短期記憶のサイズ。
- **プロンプト**: モデルに与える指示や質問の文章。
- **パラメータ**: モデル内部の数値の総数。多いほど賢いが重い（例: 数十億〜数千億）。
- **ハルシネーション**: モデルが事実でないことを自信ありげに答えてしまう現象。
- **ファインチューニング**: 既存モデルを特定用途向けに追加学習させること。

## vault 内での使われ方

- [[claude-code]] — Anthropic 公式の CLI エージェント（LLM がターミナルでコードを読み書き・実行）
- [[agentic-coding]] — LLM がコードの読み書き・実行・テストを自律的に行う開発パラダイム
- [[almide]] — LLM のコード生成向けに最適化された静的型付き言語
- [[rtk]] — LLM のトークン消費を 60-90% 削減する Rust 製 CLI プロキシ
- [[unillm]] — Anthropic / OpenAI / Groq / Gemini 等を統一する Edge 向け LLM インターフェース
- [[llm-queue-dispatcher]] — LLM リクエスト用のスコアリング型キューイングシステム
- [[llm-throttle]] — LLM API 向け RPM/TPM デュアルレート制限ライブラリ
- [[llmine]] — OpenAI/Claude/Bedrock/Ollama 等を CLI から統一的に叩くツール
- [[fuzztok]] — CJK 対応のファジー（推定）トークン数計算ライブラリ
- [[memory-rag]] — Vercel AI SDK 連携の in-memory RAG ライブラリ
- [[capto]] — マルチモーダル LLM 向けに最適化された PDF スナップショット生成 CLI
- [[fractop]] — LLM のコンテキスト上限を超える長文を分割・並列処理するチャンク処理ライブラリ
